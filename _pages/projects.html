Zion: Comprehensive, Adaptive, and Lightweight Hardware Prefetcher

Research Published

Technologies: C++, ChampSim, Memory Architecture, Hardware Design

Designed and implemented Zion — a novel L2 hardware prefetcher built around Independent Temporal-Spatial Modules (ITSM). Segmented delta and stride predictors eliminate inter-stream conflicts that degrade unified-table prefetchers under mixed workloads. Runtime feedback mechanisms — confidence-based filtering, periodic accuracy monitoring, and MSHR-aware control — dynamically regulate prefetch aggressiveness and insertion behavior under memory pressure. Hardware cost: ~9KB storage at L2 cache level.

Results vs. IPCP, Bingo, SPP, Berti: 4-core SPEC up to 43.2% speedup · self-attention workloads up to 43.0% speedup · 90.5% LLC miss coverage on LLM attention (4-core) · heterogeneous 4-core mixes 63.8% average improvement over no prefetching.

Published: Design, Automation and Test in Europe (DATE 2026)

ChampSim Multi-Core Simulation Engine — Performance Overhaul

Research Systems Programming

Technologies: C++, Multi-threading, Memory Architecture, XZ Compression

Redesigned core simulation pipeline of ChampSim — the standard academic CPU simulator for memory hierarchy and prefetcher research. Replaced shallow-copy object passing with reference-based data flow, restructured internal data layout for cache efficiency, and consolidated per-core XZ trace decompression from N independent threads into a single unified thread co-located with the simulation loop, reducing OS context switching overhead under parallel workloads.

Results: 1-core 2–3× speedup · 2-core 3× · 4-core 5× · 8/16-core >5× · 30–40% increase in concurrent simulations · up to 90% reduction in end-to-end experiment time.
